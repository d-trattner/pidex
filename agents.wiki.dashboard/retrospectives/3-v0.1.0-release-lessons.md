# Release Lessons: dashboard parity project scope and full-cycle recovery

## What went well
- Project selector, project-scoped routes, and weekly/monthly token pagination reached full scope after continuation and two fix cycles.
- Security dependency remediation removed a critical supply-chain risk before final release.
- Missing monthly token flow and project-scoped live flow were corrected before UAT closure.
- G9 layout rejection was fixed by tightening CSS cascade and strengthening the regression check.

## What went wrong
- QA tooling assumptions differed from runtime environment, causing temporary `QA Blocked` states despite later evidence.
- Initial plan and early implementation missed some plan-specified behaviors (monthly token UI, live scope, scoped page resets), causing review loops.
- Status and evidence in handoff docs drifted temporarily, requiring manual reconciliation.

## Key lessons
- Treat security findings and G9 rejections as mandatory hard-stop milestones with reconciled final evidence before further handoffs.
- Validate layout-critical gates with viewport-aware proof, not class text checks alone.
- Keep project query and token pagination contracts centralized and tested at both helper and route/API layers.
- Standardize QA/browser dependency assumptions so environment gaps do not generate false blocks.
